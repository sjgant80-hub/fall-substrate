§ ii · what fall-substrate is

A factory that researches itself.

The factory of factories was the prototype. The substrate is the destination. fall-substrate doesn't just generate a tool from a spec — it generates the spec FROM research, then verifies the result against the original research, then ships it. End-to-end, sovereign, cited.

1.

Plausibility ≠ truth.

In our deep-research test 52% of LLM-generated claims were refuted. Adversarial verification is a first-class primitive here, not an afterthought.

2.

Sovereignty ≠ regression.

Single-file tools that run on the user's device beat cloud platforms when the data is sensitive, the cost is constant, and the surface is auditable.

3.

Specs from research, not vibes.

A tool spec generated from 23 cited sources outperforms a spec dreamt up by a single LLM in one pass. Every feature traces to its evidence.

4.

Audit ≠ trust.

Every output traceable to inputs. Every claim sourced. Every transformation Ed25519-signed across prevHash → docHash → configVersion.
